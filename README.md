# claude-skills

Global [Claude Code](https://docs.anthropic.com/en/docs/claude-code) rules, skills, hooks, and commands. Symlinked into `~/.claude/` so they apply across all projects.

## Setup

```bash
ln -s /path/to/claude-skills/rules ~/.claude/rules
ln -s /path/to/claude-skills/skills ~/.claude/skills
ln -s /path/to/claude-skills/commands ~/.claude/commands
ln -s /path/to/claude-skills/hooks ~/.claude/hooks
ln -s /path/to/claude-skills/settings.json ~/.claude/settings.json
```

Claude Code loads both `~/.claude/` (global) and `<project>/.claude/` (project-local) additively.

## Structure

```
rules/                              # Auto-loaded on every conversation
  python-architecture.md            # uv, pydantic-settings, structlog, async
  python-security.md                # credentials, logging, input validation
  python-testing.md                 # 80% coverage, mock patterns, ruff
  tdd-enforcement.md                # mandatory TDD via /feature-workflow
  retrospective-discipline.md       # auto-trigger /retrospective on hard tasks

commands/                           # Invoked via /command-name
  architect.md

hooks/                              # Shell hooks for tool events
  pre-commit-verify.sh
  post-commit-retrospective.sh
  post-test-debug-reminder.sh
  session-start-stitch.sh
  stop-notify.sh

skills/                             # Invoked via /skill-name (79 skills)
  _registry.json                    # Auto-generated registry (machine-readable)
  _index.md                         # Auto-generated categorized index
  SKILL-FORMAT.md                   # YAML frontmatter schema standard
  <skill-name>/SKILL.md             # one folder per skill
```

## Skills index

Categorized listing lives in [`skills/_index.md`](skills/_index.md) and is regenerated by:

```bash
uv run --with pyyaml python scripts/generate-registry.py
```

This updates both `_registry.json` (machine-readable) and `_index.md` (human-readable) from each skill's `SKILL.md` frontmatter — including its `category:` field.

### Categories

Each `SKILL.md` declares one of:

- **think** — plan/ideate before code (deep-interview, office-hours, writing-plans, plan-{eng,design}-review, autoplan, mermaid)
- **build** — implementation (coding-general, feature-workflow, fastapi-guidelines). `/feature-workflow` is the canonical TDD+E2E entrypoint that the global `tdd-enforcement.md` rule invokes
- **debug** — debugging workflow (investigate). `/investigate` is the gstack 4-phase Iron Law debugger with freeze-hook integration
- **review** — PR / code review (review, design-review, cso, ai-slop-cleaner, verification-before-completion, iterative-subagent-review-loop)
- **test** — testing patterns (qa, benchmark, mock-testing-patterns, asyncpg-testing-patterns, playwright-e2e-patterns, pytest-* gotchas, test-against-source-contracts, mcp-python-sdk-testing)
- **ship** — release flow (ship, land-and-deploy, canary)
- **reflect** — retrospective / learning (retrospective, retro, learn, checkpoint)
- **ops** — workflow safety / orchestration (careful, freeze, dispatching-parallel-agents, orchestrate-worktree)
- **utils** — domain helpers (analysis-cost-report, bsvibe-auth-centralization, content-blog-post)
- **trap** — single-incident knowledge artifacts (≈40 skills, accumulated by `/retrospective`)

## Adjacent tooling

- **ouroboros** (multi-runtime, MCP) — `Q00/ouroboros`, installed via `uv tool install 'ouroboros-ai[all]'` then `ouroboros setup --runtime <claude|codex|opencode>`. Provides spec-first Interview → Seed → Execute → Evaluate → Evolve workflow as MCP tools. Registered for Claude in `~/.claude/mcp.json`. There is intentional concept-overlap with `/deep-interview` (skill-level Socratic clarifier) and `/writing-plans` (multi-step plan authoring); use ouroboros when you want the full ambiguity-scored loop with persistent state, use the skills for one-shot in-conversation use.
