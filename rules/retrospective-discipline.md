# Retrospective Discipline: Self-Evolution

## Core Philosophy

어려움을 겪은 경험 = 스킬 자산.
작업을 완료했다는 사실이 아니라, 과정에서 겪은 어려움과 통찰이 자산화 대상이다.

## When to Self-Initiate /retrospective

다음 중 하나라도 해당하면 작업 마무리 시점에 /retrospective를 **바로 실행**:

### 어려움 신호 (핵심 트리거)

- 처음 시도가 틀렸고 접근 방식을 바꿔야 했을 때
- 예상보다 복잡한 원인을 발견했을 때 (겉으로 보이는 것 ≠ 실제 문제)
- 공식 문서나 기존 지식이 맞지 않아서 직접 검증해야 했을 때
- 여러 번 실패 후 비자명한 해결책을 찾았을 때
- "왜 이게 안 됐지?"를 오래 고민했을 때

### 발견 신호 (가치 있는 인사이트)

- 더 나은 패턴이나 방법론을 발견했을 때
- 기존 스킬/규칙에서 빠진 중요한 케이스를 발견했을 때
- 프레임워크/도구의 예상치 못한 동작을 발견했을 때

## What Does NOT Qualify

- 처음 계획대로 순조롭게 완료된 작업
- 단순 질문/답변 (어려움 없음)
- 알고 있던 것을 그대로 적용한 경우

## How to Act

어려움 신호가 감지되면 **제안 없이 바로 실행** - 사용자 승인 불필요.
"이번 작업을 회고해 스킬로 자산화합니다." 라고 알리고 /retrospective 진행.
